## 🚀 Quick Start

```bash
# Install dependencies
npm install

# Install Playwright browsers
npx playwright install

# Set up environment
cp .env .env.local  # Edit with your credentials

# Run smoke tests (fast, ~2 min)
npx playwright test --grep @smoke

# Run all UI tests
npx playwright test tests/ui

# Run API tests only
npx playwright test tests/api

# Run with visual UI
npx playwright test --headed

# Debug mode (opens Inspector GUI)
npx playwright test --debug

# Update visual snapshots
npx playwright test tests/visual --update-snapshots

# Open HTML report
npx playwright show-report

# Run with trace on
npx playwright test --trace on
```

## 🎯 Interview Cheat Sheet

**Framework in one sentence:**  
*"A layered Playwright TypeScript framework with POM, custom fixtures for auth reuse via storage state, Faker.js test data generation, and sharded GitHub Actions CI with HTML/JUnit/Allure reporting."*

**Most important design decisions:**
1. Storage State → eliminates login from 95% of tests
2. Fixtures over beforeEach → typed, lazy, composable
3. BasePage inheritance → DRY helpers across all pages
4. TypeScript strict mode → catches bugs at compile time
5. Tagged tests (@smoke, @regression) → flexible CI targeting

See `INTERVIEW_PREP.ts` for all 30 Q&As with senior-level answers.
